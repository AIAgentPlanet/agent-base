## 一、项目概述

Agent Base 是一个面向 AI 原生时代的 Web 应用开发底座框架，旨在解决 AI 辅助开发中的三个核心问题：

1. **如何让 AI 更好地理解业务**——通过高质量、可复用的提示词库（`prompts/`）和技能定义（`skills/`）沉淀业务领域知识；
2. **如何让 AI Agent 直接调用应用能力**——基于 Model Context Protocol（MCP）标准实现服务接口（`mcp/`）和 CLI 工具（`cli/`）；
3. **如何让 AI 基于统一代码规范生成代码**——提供经过验证的基础微服务模板（`services/`）。

当前项目已落地 `services/user-service`（用户服务）和 `skills/user-service`（用户服务技能定义），并自研 **ATH（Agent Trust Handshake）协议**，为智能体与业务系统之间的可信交互提供端到端能力支撑。`prompts/`、`mcp/`、`cli/` 等模块已完成目录规划，处于持续建设中。

## 二、交互技术架构与关键能力

### 2.1 总体架构

Agent Base 围绕“入口—编排—协议—服务—数据”五个环节构建交互技术栈：

```
┌─────────────────────────────────────────────────────────────┐
│  用户/Agent 入口层                                             │
│  CLI / IDE 插件 / 多模态入口 / AI Agent                       │
└───────────────────┬─────────────────────────────────────────┘
                    │  意图理解 / 任务编排
┌───────────────────▼─────────────────────────────────────────┐
│  交互协议层                                                   │
│  MCP（Model Context Protocol）+ ATH（Agent Trust Handshake）  │
└───────────────────┬─────────────────────────────────────────┘
                    │  接口代理 / 权限校验 / 审计日志
┌───────────────────▼─────────────────────────────────────────┐
│  基础服务层                                                   │
│  user-service（用户/鉴权/OAuth/ATH 代理）                     │
└───────────────────┬─────────────────────────────────────────┘
                    │  数据持久化 / 缓存 / 链路追踪
┌───────────────────▼─────────────────────────────────────────┐
│  数据与能力层                                                 │
│  PostgreSQL / Redis / 业务 API / 文件/消息/权限服务（规划中）  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键能力

| 能力域 | 说明 | 项目实现 |
|--------|------|----------|
| 用户入口 | 统一用户身份与认证入口 | `user-service` 注册/登录/JWT |
| 任务编排 | Agent 调用业务能力的编排入口 | ATH `/api/v1/ath/proxy` |
| 工具调用 | 标准化 API、插件、MCP 服务调用 | ATH 发现文档 + Scope 权限 |
| 协议适配 | 兼容 OAuth 2.0 并扩展 Agent 认证 | ATH 协议 |
| 端云协同 | 本地 CLI/IDE 与云端服务协同 | CLI / MCP Server（规划中） |
| 数据流转 | 用户数据、上下文、记忆的安全流转 | JWT/ATH Token + 审计日志 |

## 三、交互协议与工具调用能力

### 3.1 MCP（Model Context Protocol）

Agent Base 的 `mcp/` 目录用于承载 MCP Server 实现，目标是为 AI Agent 提供标准化的上下文与工具调用能力，使 Agent 能够以低侵入方式调用用户服务、文件服务、消息通知等基础能力，不依赖特定 AI 平台。

### 3.2 ATH（Agent Trust Handshake）协议

ATH 是 Agent Base 在 `user-service` 中实现的**智能体可信交互协议**，基于 OAuth 2.0 扩展，专为 AI Agent 设计，核心能力包括：

- **Agent 身份验证**：采用 ES256 签名的 Attestation JWT，验证 Agent 身份及公钥绑定；
- **双向可信握手**：用户授权 + 服务授权双重确认，防止未经授权的 Agent 调用；
- **最小权限原则**：通过 `scope`（如 `user:read`、`user:write`、`oauth:read`）精细控制访问权限；
- **全程可追溯**：所有关键交互生成签名存证，支持事后审计与责任追溯。

ATH 完整交互流程：

```
1. Discovery    → GET  /.well-known/ath.json
2. Register     → POST /api/v1/ath/agents/register  (提交 Attestation JWT)
3. Authorize    → POST /api/v1/ath/authorize        (获取用户授权 URL)
4. User Consent → GET  /api/v1/oauth/authorize      (用户登录并授权)
5. Token        → POST /api/v1/ath/token            (用 code 换 ATH token)
6. API Call     → POST /api/v1/ath/proxy            (用 ATH token 调用 API)
7. Revoke       → POST /api/v1/ath/revoke           (注销 token)
```

### 3.3 工具调用治理机制

| 治理点 | 实现方式 |
|--------|----------|
| 工具注册 | Agent 注册时声明 providers 与 scopes |
| 参数校验 | Handler 层统一校验请求参数与路径 |
| 权限控制 | ATH Token 校验 + Scope 匹配 |
| 调用限流 | 可基于 Redis 实现 Token 级限流（规划中） |
| 审计留痕 | 关键操作记录日志，支持链路还原 |

## 四、交互风险分析与治理能力体系建设

### 4.1 主要风险与项目应对

| 风险类别 | 具体风险 | 项目治理实践 |
|----------|----------|--------------|
| 主体身份与身份互认 | Agent 身份伪造、跨平台身份传递不一致 | ATH Attestation JWT + ES256 签名 + 公钥绑定 |
| 权限越界与工具调用 | 权限过大、越权调用、敏感操作误触发 | Scope 最小权限、用户显式授权、`ath/proxy` 统一代理 |
| 数据流转与上下文污染 | Token 泄露、会话劫持、上下文被污染 | Token 有效期 24h、ATH 会话 10min TTL、JTI 重放保护 |
| 交互链路与责任边界 | 跨工具调用链路长、日志缺失、责任不清 | 全链路日志、ATH 签名存证、Redis 会话关联 |

### 4.2 接入前治理能力

- **主体身份审核**：Agent 注册需提交 Attestation JWT，服务端校验 `iss/sub`、`iat` 新鲜度、`jti` 唯一性；
- **工具准入**：通过 `providers` 与 `scopes` 声明可调用能力，未声明接口拒绝访问；
- **能力目录**：`/.well-known/ath.json` 发布可用 Provider 与 Scope 清单；
- **权限分级**：`user:read`、`user:write`、`oauth:read`、`oauth:write` 等细粒度权限；
- **接口登记**：所有 ATH 可调用的接口通过 `ath/proxy` 统一代理，禁止直接暴露内部 API。

### 4.3 运行中管控能力

- **交互请求校验**：校验 ATH Token、Scope、请求路径与方法；
- **权限控制**：Token 与所申请 Scope 匹配后方可调用；
- **敏感操作确认**：用户授权页面显式展示授权范围，用户确认后发放 `code`；
- **参数检查**：Handler 层统一校验请求参数类型与范围；
- **异常阻断**：非法 Token、越权请求、重放攻击直接返回错误码；
- **上下文隔离**：每个 ATH Session 独立存储于 Redis，10 分钟过期，用完即删。

### 4.4 事后审计与责任追溯能力

- **日志记录**：`user-service.log` 记录关键操作与异常；
- **链路还原**：通过 `client_id`、`ath_session_id`、`jti`、`code` 串联完整调用链；
- **风险复盘**：支持基于 Token 与 Session 重放分析；
- **事件处置**：提供 `/api/v1/ath/revoke` 接口即时吊销 Token。

### 4.5 测试评估与标准化能力

项目提供完整的 ATH 端到端测试脚本 `scripts/test-ath.sh`，覆盖：

- Discovery → Register → Authorize → Token → Proxy → Revoke 完整流程；
- Attestation JWT 签名与验证；
- Token 过期、重放、Scope 不匹配等异常场景。

## 五、产业实践与典型应用场景

### 5.1 工具调用与协议交互治理实践

Agent Base 的 `user-service` 可作为企业内部智能体与业务系统交互的统一认证与授权网关。AI Agent 通过 ATH 协议获得用户授权后，才能调用用户资料、OAuth 客户端管理等接口，避免 Agent 越权访问企业数据。

### 5.2 端云协同与设备交互治理实践

本地 CLI（`cli/`，规划中）与 IDE 插件可作为端侧入口，通过 ATH 与云端 `user-service` 完成可信握手，实现：

- 端侧任务分发到云端模型与工具；
- 本地代码/配置与云端用户数据的受控流转；
- 用户确认后才能执行敏感操作（如创建 OAuth 客户端、修改用户资料）。

### 5.3 行业场景交互治理实践

| 行业场景 | 应用方式 |
|----------|----------|
| 企业办公 | AI Agent 通过 ATH 代理调用用户服务，自动完成成员信息管理、权限审批 |
| 安全运营 | Agent 调用用户/权限服务前必须通过身份验证与授权，所有操作可审计 |
| 云服务 | 作为云原生应用底座，为多个微服务提供统一的 Agent 接入与治理能力 |
| 开发工具 | 通过 skills/mcp/cli 为 Cursor、Claude Code 等工具提供标准化交互接口 |

### 5.4 交互安全评测与运营实践

- **提示注入测试**：ATH 代理层对请求路径与方法进行白名单校验，防止注入；
- **越权调用测试**：通过 Scope 不匹配场景验证权限控制有效性；
- **数据泄露防护**：Token 与 Session 设置短有效期，关键数据不落地；
- **日志审计**：关键接口记录操作日志，支持安全运营分析。

## 六、发展展望与标准化建议

### 6.1 技术发展展望

Agent Base 将持续围绕以下方向演进：

- **协议化**：完善 ATH 协议，支持更多签名算法与发现机制；
- **接口化**：通过 MCP Server 将用户服务、文件服务、消息通知等能力标准化；
- **策略化**：引入策略引擎，支持基于角色、场景、风险的动态授权；
- **可信化**：强化 Agent 身份验证与行为审计，支持可信执行环境（TEE）接入；
- **自动化**：提供 CLI 与 CI/CD 集成，实现 Agent 注册、测试、部署自动化；
- **轻量化**：降低接入门槛，提供多语言 SDK 与一键接入模板。

### 6.2 产业发展展望

- **交互网关**：`user-service` 可演进为企业级 Agent 交互网关，统一接入、授权、审计；
- **工具治理**：通过 `skills/` 沉淀行业技能模板，形成工具治理目录；
- **安全评测**：基于 `scripts/test-ath.sh` 扩展交互安全评测工具集；
- **行业解决方案**：面向政务、金融、企业办公等场景输出行业化 Agent 底座；
- **生态协同**：与 MCP 社区、云厂商、终端厂商共建开放生态。

### 6.3 标准化推进建议

结合项目实践，建议在以下方向推进标准化：

1. **交互术语与参考架构**：明确 Agent、工具、网关、协议等核心概念；
2. **能力要求**：制定 Agent 身份验证、权限控制、审计追溯的最小能力集；
3. **接口协议**：推动 ATH/MCP 类协议在企业服务中的互认互通；
4. **测试方法**：建立覆盖身份、授权、调用、数据、审计的测试用例库；
5. **实践指南**：发布面向企业开发者的 Agent 交互治理落地指南；
6. **评估认证**：探索 Agent 交互安全能力评估与认证机制。
